# Showcase Repo Metadata Guide

Use this guide in the public `oyakm-showcase` repository settings.

## 1) Repository description

Suggested short description:

`Sanitized showcase of OBA platform capabilities (Flutter, PHP API, backend/admin). No source code or secrets.`

## 2) Suggested topics

- `showcase`
- `whitelabel`
- `flutter`
- `php-api`
- `mysql`
- `documentation`
- `architecture`
- `mobile-app`
- `admin-panel`

## 3) README policy checks

Ensure README clearly includes:

- Documentation-only scope.
- No source code delivery statement.
- No credentials/demo access in git.
- Link to security scope matrix.
- Link to sync workflow.

## 4) License and legal note

- Add a legal note that this repository is for showcase/documentation.
- Commercial/legal terms are controlled by signed contract documents.
- Keep "legal review required" wording on draft commercial pages.

## Turkce

Bu rehber, public `oyakm-showcase` reposunun ayarlarinda kullanilmak uzere hazirlanmistir.

## 1) Repo aciklamasi

Onerilen kisa aciklama:

`Sanitized showcase of OBA platform capabilities (Flutter, PHP API, backend/admin). No source code or secrets.`

## 2) Onerilen topic'ler

- `showcase`

- `flutter`

## 3) README politika kontrolleri

README iceriginde su maddelerin net oldugunu dogrula:

- Dokuman odakli kapsam.
- Kaynak kod teslimi olmadigi ifadesi.
- Git icinde credential/demo erisimi olmadigi.
- Security scope matrix baglantisi.
- Sync workflow baglantisi.

## 4) Lisans ve hukuki not

- Bu deponun showcase/dokumantasyon amacli oldugunu belirten hukuki not ekle.
- Ticari/hukuki kosullar imzali sozlesme dokumanlari ile yonetilir.
- Taslak ticari sayfalarda "legal review required" ifadesini koru.
